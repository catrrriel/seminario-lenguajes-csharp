## Camino Feliz — Flujo principal

### Paso 1: Autenticación
* Endpoint: `POST /api/login`
* Acción: Ingresar las credenciales del administrador, ejecutar y copiar el Token de la respuesta. Para posteriormente agregarlo al header Authorization y asi habiliar los endpoints protegidos.

### Paso 2: Crear un Expediente
* Endpoint: `POST /api/expedientes`
* Acción: Enviar un JSON con una carátula válida (string). Devuelve HTTP 200 OK y el ID del Expediente dado de alta con el estado inicializado en `RecienIniciado = 0`.

### Paso 3: Agregar un Trámite
* Endpoint: `POST /api/tramites`
* Acción: Asociar un trámite al expediente creado, por ejemplo, con etiqueta `Resolucion = 4`

### Paso 4: Verificar cambio de estado automático
* Endpoint: `GET /api/expedientes`
* Acción: Comprobar que el estado del expediente cambio de `RecienIniciado = 0` a `ConResolucion = 2`.

### Paso 5: Obtener los usuarios registrados
* Endpoint: `GET /api/usuarios`
* Acción: Ejecutar la petición asegurándose que esté el header Authorization con el token válido. Copiar el ID del `UsuarioConPermiso` 

### Paso 6: Modificar los permisos de un usuario
* Endpoint: `PUT api/usuarios/{id}/permisos`
* Acción: Pegar el ID como parámetro y enviar un JSON con los valores de los permisos que se quieran añadir:

| Permisos  | Valores numéricos |
| ------------- |:-------------:|
| ExpedienteAlta | 0 |
| ExpedienteBaja | 1 |
| ExpedienteModificacion | 2 |
| TramiteAlta | 3 |
| TramiteBaja | 4 |
| TramiteModificacion | 5 |

> Después de agregar algunos permisos también probar que los permisos que no integren el JSON son eliminados de los permisos ya otorgados.
